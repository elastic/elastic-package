# URL Validator with HTTP Request Validation

## Summary

Modify [`internal/llmagent/docagent/agents/urlvalidator.go`](internal/llmagent/docagent/agents/urlvalidator.go) to validate URLs by making actual HTTP HEAD requests instead of using LLM calls, while maintaining the same agent interface (`SectionAgent`) and output format (`URLCheckResult`).

## Changes

### 1. Remove LLM-based path in `Build` method

- Remove the conditional that creates an `llmagent.New` when `cfg.Model != nil`
- Always use the programmatic agent via `agent.New` with the `run` method
- Remove `google.golang.org/adk/agent/llmagent` import

### 2. Enhance `validateURLs` to make HTTP requests

- Create an HTTP client with sensible timeout (e.g., 10 seconds per request)
- Use HTTP HEAD requests (lighter than GET) to check URL accessibility
- Categorize responses:
  - **Valid**: 2xx status codes
  - **Invalid**: 4xx/5xx status codes, connection errors, timeouts
  - **Warnings**: Redirects (3xx), localhost URLs, placeholder URLs

### 3. Add concurrency for efficiency

- Process URLs concurrently using goroutines with bounded parallelism
- Use `sync.WaitGroup` and a semaphore channel to limit concurrent requests (e.g., 5 parallel)
- Respect context cancellation

## Key Implementation Details

```go
func (u *URLValidatorAgent) checkURL(ctx context.Context, client *http.Client, url string) (valid bool, warning string) {
    req, err := http.NewRequestWithContext(ctx, http.MethodHead, url, nil)
    if err != nil {
        return false, ""
    }
    resp, err := client.Do(req)
    if err != nil {
        return false, ""
    }
    defer resp.Body.Close()
    
    if resp.StatusCode >= 200 && resp.StatusCode < 300 {
        return true, ""
    }
    if resp.StatusCode >= 300 && resp.StatusCode < 400 {
        return true, fmt.Sprintf("URL redirects: %s -> %s", url, resp.Header.Get("Location"))
    }
    return false, ""
}
```
